# ============================================================================
# KEY FEATURES
# ============================================================================

DESIGNED FOR REUSABILITY ACROSS ALL PHASES:

✅ Single function to create any environment variant
   create_env("discrete", "min_steps")
   create_env("discrete", "min_fuel")
   create_env("continuous", "linear_cost")
   create_env("continuous", "quadratic_cost")

✅ Agent interface works with ANY learning algorithm
   - Tabular methods (Q-learning, SARSA, MC)
   - Deep RL (DQN, DDQN)
   - Policy gradient (REINFORCE, Actor-Critic)
   - Custom methods (just inherit BaseAgent)

✅ Generic training loop accepts any agent, no duplicated code
   train_agent(agent, env, ...) works for all methods

✅ Multi-seed statistical rigor built-in
   - Aggregate metrics across seeds
   - Confidence intervals computed automatically
   - Convergence analysis standardized

✅ Configurable via arguments 
   - State discretization: n_pos_bins, n_vel_bins
   - Training: n_episodes, learning_rate, gamma
   - Evaluation: n_eval_episodes
   - All easily tweakable

✅ Tensorboard integration for live monitoring
   - Automatic logging during training
   - Metrics tracked per episode
   - View plots in real-time

# ============================================================================
# PHASE-BY-PHASE APPLICATION
# ============================================================================

PHASE 1: Environment & Baselines
└─ Create env with StateDiscretizer
   Implement QLearning(BaseAgent)
   Train with train_agent()
   Visualize policies & learning curves

PHASE 2: Reward Scenarios
├─ Loop over scenario in ["min_steps", "min_fuel", ...]
├─ create_env() with each scenario
├─ Train same agent
└─ store_results & compare metrics

PHASE 3: Discretization
├─ Loop over n_bins in [5, 10, 15, 20, 30, 40]
├─ StateDiscretizer(n_bins, n_bins)
├─ Train Q-learning
└─ Analyze convergence vs bin count

PHASE 4: Tabular Methods
├─ Implement: SARSA(BaseAgent), MonteCarlo(BaseAgent)
├─ For each method in [QLearning, SARSA, MC]:
│   └─ train_agent(agent, ...)
└─ Compare: success rate, convergence, stability

PHASE 5-6: Deep RL & Policy Gradient
├─ Implement: DQN(BaseAgent), PolicyGradient(BaseAgent)
├─ Same training loop (no code changes!)
├─ train_agent(agent, ...)
└─ Compare with tabular methods

PHASE 7: Policy Interpretation
├─ Use get_hyperparams(), extract policies
├─ Analyze policy structure
├─ Compare policies across methods
└─ Physical interpretation

PHASE 8: Comparative Analysis
├─ Aggregate all metrics with StatisticalAnalyzer
├─ Compute statistics across seeds
├─ Generate comparison tables
└─ Create visualizations
